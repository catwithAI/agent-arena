# AtCoder 广告投放优化题

## 题目描述

AtCoder 需要在首页为 n 家公司放置广告。广告区域是一个 10000 x 10000 的正方形网格。

第 i 家公司希望获得一个矩形广告区域。这个矩形必须包含点 `(x_i + 0.5, y_i + 0.5)`，并且面积尽量接近期望面积 `r_i`。你需要为所有公司确定广告矩形，使总满意度尽可能高。

![题目可视化示例](assets/visualization.png)

## 约束

- 每个广告区域必须是边与坐标轴平行、顶点坐标为整数、面积为正的矩形
- 不同矩形可以共享边，但不能有重叠面积
- 可以留空白区域
- `50 <= n <= 200`
- `0 <= x_i, y_i <= 9999`，且所有 `(x_i, y_i)` 两两不同
- `r_i >= 1`，且 `sum(r_i) = 10^8`

## 评分公式

第 i 家公司的满意度 `p_i`：

- 如果广告区域不包含点 `(x_i + 0.5, y_i + 0.5)`，则 `p_i = 0`
- 如果广告区域包含该点，且矩形面积为 `s_i`，则
  `p_i = 1 - (1 - min(r_i, s_i) / max(r_i, s_i))^2`

单个测试用例得分：

```text
round(10^9 * sum(p_i) / n)
```

目标是最大化所有固定评测用例的总分。

## 输入格式

程序从标准输入读取：

```text
n
x_0 y_0 r_0
x_1 y_1 r_1
...
x_{n-1} y_{n-1} r_{n-1}
```

## 输出格式

程序向标准输出写入：

```text
a_0 b_0 c_0 d_0
a_1 b_1 c_1 d_1
...
a_{n-1} b_{n-1} c_{n-1} d_{n-1}
```

其中 `(a_i, b_i)` 和 `(c_i, d_i)` 是第 i 家公司广告矩形的一组对角顶点。

必须满足：

```text
0 <= a_i < c_i <= 10000
0 <= b_i < d_i <= 10000
```

## 评测方式

评测由两部分组成：**本地调试** 和 **最终批量评测**。

### 本地调试

`tools/` 提供输入生成器和评分程序，可用于自由调试：

```bash
# 生成测试输入
printf '0\n' > /tmp/seeds.txt
rm -rf /tmp/ad_cases && mkdir -p /tmp/ad_cases
./tools/bin/gen /tmp/seeds.txt -d /tmp/ad_cases
cp /tmp/ad_cases/0000.txt input.txt

# 运行你的程序
./my_solution < input.txt > output.txt

# 本地评分
./tools/bin/tester input.txt output.txt
# stderr 输出：Score = <score>
```

`tools/bin/gen` 会读取 seed 文件，并把生成的测试用例写入目标目录。每行一个 seed，空行会被忽略。

例如：

```bash
seq 0 9 > /tmp/seeds.txt
./tools/bin/gen /tmp/seeds.txt -d /tmp/ad_cases
```

这会生成 10 个本地测试用例。更多细节见 `tools/README.md`。

### 最终批量评测

最终评分使用仓库内固定的 50 个评测用例。scorer 会逐个运行你的 `solution.cpp`，校验输出是否合法，并累加每个用例的原始得分。

每次提交后，系统会返回：

- 每个用例的运行状态：通过、编译错误、运行错误、超时或输出非法
- 每个用例的得分
- 50 个用例的总分

最终 `batch_score` 会把原始总分按如下方式归一化到 0-100：

```text
round(100 * total_score / (cases * 1_000_000_000))
```

## 输入生成方式

- `n = round(50 * 4^rand())`，其中 `rand()` 是 `[0, 1)` 上的均匀随机数
- `(x_i, y_i)` 从 `{0, ..., 9999}^2` 中无放回采样，且两两不同
- `r_i` 通过从 `{1, ..., 99999999}` 中无放回采样 `n-1` 个切分点生成，并保证 `sum(r_i) = 10^8`

`n` 更偏向较小值：接近 50 的概率最高，接近 200 的概率最低。`r_i` 的差异可能很大。

## 解题提示

这是一个 NP-hard 的矩形放置优化问题。可以考虑：

- 贪心策略：按某种优先级逐个放置矩形
- 局部搜索或模拟退火：从一个可行解出发做邻域搜索
- 分治：递归划分平面区域后在子区域内放置矩形
- 特别注意硬约束：矩形不能重叠
- 充分使用本地工具调试，再提交最终版本
