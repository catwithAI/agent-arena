# Apple Incremental Game -- 机器生产优化

## 题目描述

APPLE ARTIS 公司开发了一套用于批量生产苹果的分层机器系统。系统共有 `N x L` 种机器，由 `N` 种机器 ID 和 `L` 个等级组成。等级为 `i`、ID 为 `j` 的机器记作机器 `j^i`，其中 `0 <= i < L`，`0 <= j < N`。

机器 `j^0` 的生产能力为 `A_j`。机器 `j^i` 的初始强化成本为 `C_{i,j}`。

你的目标是在 `T` 回合结束后最大化苹果总数。

## 生产计划流程

令 `B_{i,j}` 表示机器 `j^i` 的数量，初始全为 `1`；令 `P_{i,j}` 表示机器 `j^i` 的强化等级，初始全为 `0`。初始苹果数为 `K`。

每一回合按以下顺序执行：

1. **行动阶段**：选择以下操作之一：
   - 强化机器 `j^i`：消耗 `C_{i,j} x (P_{i,j} + 1)` 个苹果，并使 `P_{i,j}` 增加 `1`。如果强化后苹果数会变成负数，则该操作非法。
   - 什么都不做。

2. **生产阶段**：按等级 `0, 1, 2, 3` 的顺序执行：
   - 对等级 0 机器：苹果数增加 `A_j x B_{i,j} x P_{i,j}`。
   - 对等级 1 及以上机器：`B_{i-1,j}` 增加 `B_{i,j} x P_{i,j}`。

## 约束

- `N = 10, L = 4, T = 500, K = 1`
- `1 <= A_j <= 100`，且 `A` 按升序排列
- `1 <= C_{i,j} <= 1.25 x 10^12`
- `C_{0,0} = 1`

## 评分公式

令 `S` 为 `T` 回合结束后的苹果数。你的单个用例得分为：

```text
score = round(100000 x log2(S))
```

目标是最大化得分，也就是最大化最终苹果数。

以下情况会判为错误答案：

- 强化操作导致苹果数变成负数
- 指定不存在的机器等级或 ID
- 输出少于 `T` 个动作

## 输入格式

从标准输入读取：

```text
N L T K
A_0 A_1 ... A_{N-1}
C_{0,0} C_{0,1} ... C_{0,N-1}
C_{1,0} C_{1,1} ... C_{1,N-1}
...
C_{L-1,0} C_{L-1,1} ... C_{L-1,N-1}
```

## 输出格式

向标准输出输出恰好 `T` 行：

- 强化机器 `j^i`：输出 `i j`
- 什么都不做：输出 `-1`

允许输出以 `#` 开头的注释行。

## 评测方式

评测分为本地测试和最终评测。

### 本地测试

`tools/` 提供输入生成器和评分程序：

```bash
# 生成测试输入
./tools/bin/gen <seed> > input.txt

# 运行你的解法
python3 solution.py < input.txt > output.txt

# 评分
./tools/bin/tester input.txt output.txt
```

可以使用 `0..10000` 范围内任意 seed 生成本地测试用例。你可以在本地生成任意多用例用于调试和优化。

不要使用 `tools/src/verifier.py` 作为评分依据，它只检查输出格式，不模拟游戏，也不计算得分。

### 最终评测

最终分数由 50 个隐藏固定测试用例决定。每个用例调用你的 `solution.py`，再用官方 `tester` 校验并计分。最终原始分数为 50 个用例分数之和。

## 输入生成规则

- `N = 10, L = 4, T = 500, K = 1`
- `A_0 = 1`
- 对 `j != 0`：`A_j = round(10^rand_double(0, 2))`，然后整体升序排序
- `C_{0,0} = 1`
- 其他成本：`C_{i,j} = round(A_j x 500^i x 10^rand_double(0, 2))`，上限为 `10^15`

## 解题提示

这是一个资源分配和增量游戏优化问题：

- 高等级机器会生产低等级机器，形成复利式增长
- 早期强化便宜且回报周期长的机器通常更有价值
- 需要在直接产苹果的等级 0 机器和带来乘数增长的高等级机器之间权衡
- 强化成本会随当前强化等级线性增加：`C_{i,j} x (P_{i,j} + 1)`

可尝试的方向：

- 贪心：每回合选择投资回报率最高的强化
- 结合剩余回合数估算长期收益
- 用模拟或局部搜索比较不同强化序列
